- 所有 API 响应遵循统一包装格式：成功返回 `{ "data": ... }`，错误返回 `{ "error": { "code": ..., "message": ... } }`。
- 数据库操作函数统一接收 `&SqlitePool` 作为第一个参数，并返回 `Result<T, sqlx::Error>` 或自定义 `AppError`。
- 后台服务模块（Parser/Filter/Pusher）均实现为 `start_xxx_loop` 无限循环函数，内部包含固定的休眠间隔与错误捕获逻辑。
- 使用 `tracing` 宏进行结构化日志记录，关键业务流程（如采集、检测、推送）均包含 info 级别的进度日志和 error 级别的异常日志。
