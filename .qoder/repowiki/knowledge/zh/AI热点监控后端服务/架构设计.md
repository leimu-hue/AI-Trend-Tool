- **分层架构**：采用经典的 `handlers` (路由处理) -> `middleware` (中间件/认证) -> `services` (业务逻辑) -> `db` (数据访问) -> `models` (数据模型) 的分层设计。
- **状态管理**：通过 `AppState` 结构体在 Axum 路由器中共享 `SqlitePool` 数据库连接池和 `AppConfig` 配置信息。
- **模块化组织**：`src/db` 和 `src/models` 按业务实体（如 `token`, `article`, `source`）拆分为子模块，实现关注点分离。
- **错误处理**：定义统一的 `AppError` 枚举并实现 `IntoResponse`，将数据库错误或业务逻辑错误转换为标准化的 JSON 响应格式。
- **启动引导**：`main.rs` 负责解析 CLI 参数、加载 TOML 配置、初始化数据库连接池、执行 SQL 迁移以及引导初始管理员 Token。
