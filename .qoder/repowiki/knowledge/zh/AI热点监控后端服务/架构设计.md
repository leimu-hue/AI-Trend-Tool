- **分层架构**：采用标准的 `src/` 目录结构，划分为 `handlers` (API入口)、`services` (业务逻辑)、`db` (数据访问) 和 `models` (数据模型)。
- **管道模式 (Pipeline)**：核心业务由 `Parser` (采集)、`Filter` (过滤/检测)、`Pusher` (推送) 三个独立后台模块组成，在 `main.rs` 中通过 `tokio::spawn` 并发运行。
- **路由与中间件**：使用 `axum` 框架，在 `routes.rs` 中定义 RESTful 接口，并通过 `middleware/auth.rs` 实现基于 Bearer Token 的认证拦截。
- **统一错误处理**：通过 `error.rs` 定义 `AppError` 枚举并实现 `IntoResponse`，确保 API 返回统一的 JSON 错误格式。
- **配置驱动**：使用 `config.toml` 和 `clap` 解析运行时参数，支持灵活切换运行模式（all/api/parser/filter/pusher）。
